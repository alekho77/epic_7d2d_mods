[size=6][b]Reduced Looting XP[/b][/size]

[size=5][b]Description[/b][/size]

Reduces XP gained from opening untouched loot containers to [b]20% of vanilla[/b], rebalancing progression toward combat and survival gameplay.

In vanilla 7 Days to Die, opening an untouched loot container awards XP proportional to the player's current game stage. Both world-placed containers (crates, safes, fridges) and zombie drop bags award XP on first open — bags spawn untouched and use the same XP code path as world containers. Players who focus on aggressive POI runs, blood moon looting, and systematic container farming can level disproportionately fast compared to players who progress mainly through combat. This mod reduces container XP while leaving kill XP, loot tables, and bag spawning behavior unchanged.

[quote]
[color=#55aa55][size=4][b]Server-Side Friendly[/b][/size][/color]

[b]If you are running a dedicated server, this mod only needs to be installed on the server.[/b]
Players connecting to the server [b]do not need to download or install anything[/b] on their game clients — all changes are synced automatically when they join.
[/quote]

[size=5][b]Features[/b][/size]

[list]
[*]Reduces untouched container XP (world containers and zombie drop bags) to 20% of vanilla — effective rate changes from [b]gameStage × (XPMultiplier/100)[/b] to [b]gameStage × (XPMultiplier/100) × 0.2[/b]
[*]The looting XP branch stays enabled — it is rebalanced, not removed
[*]All other XP sources (kills, crafting, quests, selling, harvesting, upgrading) remain completely unchanged
[*]Loot quality, loot quantity, scavenging speed, and Lucky Looter perk are [b]not[/b] affected
[*]Zombie drop bags ([b]EntityLootContainer*[/b]) spawn untouched and award first-open XP the same as world containers — this mod reduces that XP too
[*]Player chests, secure storage, and dropped player backpacks are excluded from looting XP by the game engine and are unaffected by this mod
[/list]

[size=5][b]How It Works[/b][/size]

The vanilla XP formula for opening an untouched container is:

[quote][b]xp = floor(gameStage × (XPMultiplier / 100))[/b][/quote]

[b]XPMultiplier[/b] is the server's XP Multiplier game setting (default [b]100[/b] — meaning 100%). With default settings, vanilla awards [b]gameStage[/b] XP per untouched container — e.g., game stage 50 gives 50 XP. Both world-placed containers and zombie drop bags use this same code path.

This mod applies [b]perc_add -0.8[/b] to [b]PlayerExpGain[/b] for the [b]Looting[/b] tag. This reduces the XP multiplier by 80%, leaving 20% of vanilla:

[quote][b]xp = floor(gameStage × (XPMultiplier / 100) × (1 + (-0.8)))[/b]
[b]    = floor(gameStage × (XPMultiplier / 100) × 0.2)[/b][/quote]

With default server settings ([b]XPMultiplier = 100[/b]):

[quote][b]xp = floor(gameStage × 0.2)[/b][/quote]

The result is [b]20% of vanilla XP[/b] at any given game stage. XP bonus buffs such as Grandpa's Learning Elixir ([b]perc_add 0.2[/b]) stack additively with this mod's [b]-0.8[/b], raising the effective multiplier from [b]0.2[/b] to [b]0.4[/b]. This keeps XP boosts proportional and does not restore the full vanilla rate.

[size=4][b]Important to Keep in Mind[/b][/size]

Looting XP is an integer — it is floor-truncated. At very low game stage the result rounds down to [b]0 XP[/b] per container.

With default server settings ([b]XPMultiplier = 100[/b]), the first game stage that produces at least [b]1 XP[/b] per untouched container:

[list]
[*][b]-0.90[/b] — 10% of vanilla — first non-zero at game stage [b]10[/b]
[*][b]-0.80[/b] (this mod) — 20% of vanilla — first non-zero at game stage [b]5[/b]
[*][b]-0.75[/b] — 25% of vanilla — first non-zero at game stage [b]4[/b]
[*][b]-0.50[/b] — 50% of vanilla — first non-zero at game stage [b]2[/b]
[*][b]0[/b] (vanilla) — 100% — first non-zero at game stage [b]1[/b]
[/list]

With default server settings, this mod produces non-zero looting XP from game stage 5 onward, consistently at 20% of the vanilla rate throughout the game.

[quote][b]Note:[/b] If the server's [b]XPMultiplier[/b] is set below 100 (e.g., 20 for a 20% global XP rate), the threshold shifts higher and the absolute XP values scale down proportionally. For example, with [b]XPMultiplier = 20[/b] the first non-zero GS with this mod is 25.[/quote]

[size=4][b]Companion Mod[/b][/size]

Works well alongside [url=https://github.com/alekho77/epic_7d2d_mods][b]EV_RemoveTraderXP[/b][/url], which completely removes XP from selling items to traders. Together, the two mods close the two most common non-combat XP exploits and shift the progression curve decisively toward zombie killing, crafting, and questing.
