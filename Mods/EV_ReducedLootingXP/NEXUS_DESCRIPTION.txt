[size=6][b]Reduced Looting XP[/b][/size]

[size=5][b]Description[/b][/size]

Reduces XP gained from opening untouched loot containers to [b]20% of vanilla[/b], rebalancing progression toward combat and survival gameplay.

In vanilla 7 Days to Die, the active first-open looting XP branch awards XP equal to the player's current game stage when an untouched world container is opened. Players who focus on aggressive POI routes, airdrops, and other untouched containers can level disproportionately fast compared to players who mainly progress through direct combat. Zombie drop bags use a separate entity-loot path, but vanilla spawns them prefilled and already touched, so that bag-side first-open XP branch is effectively disabled by default. This mod reduces the active container XP branch while leaving kill XP, loot tables, and bag spawning behavior unchanged.

[quote]
[color=#55aa55][size=4][b]Server-Side Friendly[/b][/size][/color]

[b]If you are running a dedicated server, this mod only needs to be installed on the server.[/b]
Players connecting to the server [b]do not need to download or install anything[/b] on their game clients — all changes are synced automatically when they join.
[/quote]

[size=5][b]Features[/b][/size]

[list]
[*]Reduces untouched world-container XP to 20% of vanilla (game stage × 0.2 instead of game stage × 1)
[*]The active looting XP branch stays enabled — it is rebalanced, not removed
[*]All other XP sources (kills, crafting, quests, selling, harvesting, upgrading) remain completely unchanged
[*]Loot quality, loot quantity, scavenging speed, and Lucky Looter perk are [b]not[/b] affected
[*]Zombie drop bags keep vanilla behavior: they are separate [b]EntityLootContainer*[/b] drops and their bag-side first-open XP branch remains effectively disabled because they spawn already touched
[*]Player chests, secure storage, and dropped player backpacks are excluded from looting XP by the game engine and are unaffected by this mod
[/list]

[size=5][b]How It Works[/b][/size]

Vanilla has two separate loot-related XP paths:

[list=1]
[*]Untouched world containers use the active looting XP branch. On first open, the game grants XP equal to the player's current [b]gameStage[/b], then applies [b]PlayerExpGain[/b] with the [b]Looting[/b] tag.
[*]Zombie drop bags are separate [b]EntityLootContainer*[/b] entities. The game spawns them with copied inventory content, which marks their backing loot container as already touched before the player opens it. Because the first-open looting XP award only fires for untouched containers, this bag-side branch is effectively disabled in vanilla.
[/list]

This mod changes only the active container branch by setting [b]PlayerExpGain[/b] base multiplier to [b]0.2[/b] for the [b]Looting[/b] tag on the player entity. Using [b]base_set 0.2[/b] scales the base before any additive percentage buffs apply — so a player with Grandpa's Learning Elixir (+20%) gains [b]gameStage × 0.2 × 1.2[/b] rather than [b]gameStage × 1.2[/b], keeping XP boost items proportional without restoring the full vanilla looting XP.

[size=4][b]Important to Keep in Mind[/b][/size]

Looting XP from the active untouched-container branch is granted as whole numbers. At low game stage, strong reductions can round down to [b]0 XP[/b] per container.

With no XP buffs or perks active, players start receiving at least [b]1 XP[/b] from untouched world containers at these game stages:

[list]
[*][b]0.10[/b] — first non-zero loot XP at game stage [b]10[/b]
[*][b]0.20[/b] — first non-zero loot XP at game stage [b]5[/b]
[*][b]0.25[/b] — first non-zero loot XP at game stage [b]4[/b]
[*][b]0.50[/b] — first non-zero loot XP at game stage [b]2[/b]
[*][b]1.00[/b] — first non-zero loot XP at game stage [b]1[/b]
[/list]

That is why this mod uses [b]0.2[/b] instead of [b]0.1[/b]: it still heavily nerfs looting XP but avoids making early-game container XP disappear entirely.

[size=5][b]Installation[/b][/size]

[list=1]
[*]Copy the [b]EV_ReducedLootingXP[/b] folder into your [b]7 Days to Die/Mods/[/b] directory.
[*]Restart the game or server.
[/list]

[size=5][b]Compatibility[/b][/size]

[list]
[*]7 Days to Die 1.0 (Alpha 21+)
[*]Server-side mod — works without client installation
[*]May conflict with mods that modify [b]PlayerExpGain[/b] on the [b]playerMale[/b] entity class
[/list]

[size=4][b]Companion Mod[/b][/size]

Works well alongside [url=https://github.com/alekho77/epic_7d2d_mods][b]EV_RemoveTraderXP[/b][/url], which completely removes XP from selling items to traders. Together, the two mods close the two most common non-combat XP exploits and shift the progression curve decisively toward zombie killing, crafting, and questing.
