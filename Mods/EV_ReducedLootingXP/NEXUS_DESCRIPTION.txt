[size=6][b]Reduced Looting XP[/b][/size]

[quote][color=#cc9900][b]Test Build[/b][/color] — exact XP reduction is under active investigation.[/quote]

[size=5][b]Description[/b][/size]

Reduces XP gained from opening untouched loot containers, rebalancing progression toward combat and survival gameplay.

In vanilla 7 Days to Die, opening an untouched loot container awards XP proportional to the player's current game stage. Both world-placed containers (crates, safes, fridges) and zombie drop bags award XP on first open — bags spawn untouched and use the same XP code path as world containers. Players who focus on aggressive POI runs, blood moon looting, and systematic container farming can level disproportionately fast compared to players who progress mainly through combat. This mod reduces container XP while leaving kill XP, loot tables, and bag spawning behavior unchanged.

[quote]
[color=#55aa55][size=4][b]Server-Side Friendly[/b][/size][/color]

[b]If you are running a dedicated server, this mod only needs to be installed on the server.[/b]
Players connecting to the server [b]do not need to download or install anything[/b] on their game clients — all changes are synced automatically when they join.

[size=5][b]Features[/b][/size]

[list]
[*]Reduces XP from opening untouched containers (world containers and zombie drop bags)
[*]The looting XP branch stays enabled — it is rebalanced, not removed
[*]All other XP sources (kills, crafting, quests, selling, harvesting, upgrading) remain completely unchanged
[*]Loot quality, loot quantity, scavenging speed, and Lucky Looter perk are [b]not[/b] affected
[*]Zombie drop bags ([b]EntityLootContainer*[/b]) spawn untouched and award first-open XP the same as world containers — this mod reduces that XP too
[*]Player chests, secure storage, and dropped player backpacks are excluded from looting XP by the game engine and are unaffected by this mod
[/list]

[size=5][b]How It Works[/b][/size]

Confirmed from decompiling [b]Assembly-CSharp.dll[/b] ([b]XUiC_LootWindowGroup[/b], [b]Progression.AddLevelExp[/b], [b]EffectManager.GetValue[/b]):

[list]
[*]Looting XP is awarded on first open of an untouched non-player container
[*]The base XP value passed to the effect system is [b]gameStage × (XPMultiplier / 100)[/b]
[*][b]EffectManager.GetValue[/b] returns [b]_base_value × _perc_value[/b]
[*][b]base_set VALUE[/b] replaces [b]_base_value[/b] with the literal value (absolute replacement)
[*][b]perc_add VALUE[/b] accumulates into [b]_perc_value[/b] (which starts at 1.0)
[*]The final result is cast to [b]int[/b] (floor-truncated before granting)
[/list]

This mod applies [b]base_set 0.8[/b] to [b]PlayerExpGain[/b] for the [b]Looting[/b] tag. [b]The exact XP output of this configuration is under investigation[/b] — in-game testing at known game stages is required to confirm behavior.

[quote][b]Note:[/b] The Project Z overhaul mod applies [b]PlayerExpGain perc_add -1 tags="Looting"[/b] via an always-active buff, which zeroes looting XP on its own. Test results from Project Z sessions may not reflect vanilla behavior.[/quote]

[size=5][b]Installation[/b][/size]

[list=1]
[*]Copy the [b]EV_ReducedLootingXP[/b] folder into your [b]7 Days to Die/Mods/[/b] directory.
[*]Restart the game or server.
[/list]

[size=5][b]Compatibility[/b][/size]

[list]
[*]7 Days to Die 1.0 (Alpha 21+)
[*]Server-side mod — works without client installation
[*]May conflict with mods that modify [b]PlayerExpGain[/b] on the [b]playerMale[/b] entity class
[*][b]Project Z:[/b] The Z_Game_Balance sub-mod applies its own [b]PlayerExpGain perc_add -1 tags="Looting"[/b] modifier, which stacks with this mod. Behavior under Project Z differs from vanilla.
[/list]

[size=4][b]Companion Mod[/b][/size]

Works well alongside [url=https://github.com/alekho77/epic_7d2d_mods][b]EV_RemoveTraderXP[/b][/url], which completely removes XP from selling items to traders. Together, the two mods close the two most common non-combat XP exploits and shift the progression curve decisively toward zombie killing, crafting, and questing.
